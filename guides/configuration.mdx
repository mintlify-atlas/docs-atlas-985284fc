Neru uses TOML for configuration. This guide covers all available options with explanations, use cases, and examples.

## Configuration File Locations

Neru loads configuration from the following locations (in order of priority):

1. **`$XDG_CONFIG_HOME/neru/config.toml`** (if `XDG_CONFIG_HOME` is set)
2. **`~/.config/neru/config.toml`** (recommended)
3. **`~/.neru.toml`** (legacy)
4. **`neru.toml`** (current directory)
5. **`config.toml`** (current directory)

This search order is the same on all platforms (macOS, Linux, Windows).

### Global Flag

Use the `--config` or `-c` flag to specify a custom config file path when launching:

```bash
neru launch --config /path/to/config.toml
```

**No config file?** Neru works out of the box with sensible defaults. Copy from [default-config.toml](https://github.com/y3owk1n/neru/blob/main/configs/default-config.toml) to customize.

**Reload config changes:**

```bash
neru config reload
```

Or restart the daemon:

```bash
pkill neru && neru launch
```

## General Settings

Core behavior settings that affect all Neru functionality.

<ParamField path="general.excluded_apps" type="array" default='[]'>
  Bundle IDs where Neru won't activate. Use this to prevent Neru from activating in specific applications like terminals or screen sharing apps.

  <CodeGroup>
  ```toml Example
  [general]
  excluded_apps = [
      "com.apple.Terminal",
      "com.googlecode.iterm2",
      "com.microsoft.rdc.macos"
  ]
  ```
  </CodeGroup>

  Find bundle IDs using: `osascript -e 'id of app "Safari"'`
</ParamField>

<ParamField path="general.accessibility_check_on_start" type="boolean" default="true">
  Verify accessibility permissions on launch. Set to `false` to skip the check (may cause runtime errors).
</ParamField>

<ParamField path="general.restore_cursor_position" type="boolean" default="false">
  Return cursor to where it was before entering a navigation mode. Mutually exclusive with `center_cursor_position`.
</ParamField>

<ParamField path="general.center_cursor_position" type="boolean" default="false">
  Center cursor on the current screen when exiting a navigation mode. Mutually exclusive with `restore_cursor_position`.
</ParamField>

<ParamField path="general.kb_layout_to_use" type="string" default='""'>
  Optional InputSourceID for layout mapping. When empty, Neru auto-selects the first available layout.

  <CodeGroup>
  ```toml macOS Example
  [general]
  kb_layout_to_use = "com.apple.keylayout.ABC"
  ```
  </CodeGroup>

  Use `defaults read com.apple.HIToolbox AppleEnabledInputSources` to find available layout IDs on macOS.
</ParamField>

<ParamField path="general.mode_exit_keys" type="array" default='["Escape"]'>
  Keys that exit any active mode. This array cannot be empty - at least one exit key must be defined.

  <CodeGroup>
  ```toml Example
  [general]
  mode_exit_keys = ["Escape", "q"]
  ```
  </CodeGroup>
</ParamField>

<ParamField path="general.passthrough_unbounded_keys" type="boolean" default="false">
  Allow unbound Cmd/Ctrl/Alt shortcuts to reach macOS while a mode is active. Examples that can pass through: `Cmd+Tab`, `Cmd+Space`, `Cmd+W`.
</ParamField>

<ParamField path="general.passthrough_unbounded_keys_blacklist" type="array" default='[]'>
  Shortcuts to keep consumed while passthrough is enabled. Each entry must include at least one modifier (Cmd, Ctrl, Alt, or Option).

  <CodeGroup>
  ```toml Example
  [general]
  passthrough_unbounded_keys = true
  passthrough_unbounded_keys_blacklist = ["Cmd+W", "Cmd+Q"]
  ```
  </CodeGroup>
</ParamField>

<ParamField path="general.hide_overlay_in_screen_share" type="boolean" default="false">
  Hide Neru overlays during screen sharing. Uses macOS `NSWindow.sharingType` API. Reliability varies by macOS version and conferencing software.
</ParamField>

## Hotkeys

Global hotkeys trigger Neru navigation modes from anywhere.

<Note>
  Defining **any** custom hotkey **replaces ALL defaults**. You must explicitly define every hotkey you want.
</Note>

<ParamField path="hotkeys" type="object">
  Global hotkey definitions. Format: `"Modifier1+Modifier2+Key" = "action"`

  <CodeGroup>
  ```toml Basic Setup
  [hotkeys]
  "Cmd+Shift+Space" = "hints"
  "Cmd+Shift+G" = "grid"
  "Cmd+Shift+C" = "recursive_grid"
  "Cmd+Shift+S" = "scroll"
  ```

  ```toml With Auto-Click
  [hotkeys]
  "Cmd+Shift+Space" = "hints --action left_click"
  "Cmd+Shift+G" = "grid --action left_click"
  "Cmd+Shift+S" = "scroll"
  ```

  ```toml Shell Commands
  [hotkeys]
  "Cmd+Alt+T" = "exec open -a Terminal"
  "Cmd+Alt+C" = "exec open -a 'Visual Studio Code'"
  ```
  </CodeGroup>
</ParamField>

## Hints Mode Configuration

Hint mode uses macOS Accessibility APIs to identify clickable UI elements and overlay short labels on them.

### Basic Settings

<ParamField path="hints.enabled" type="boolean" default="true">
  Enable or disable hints mode globally.
</ParamField>

<ParamField path="hints.auto_exit_actions" type="array" default='[]'>
  Actions that auto-exit hints mode after execution. See Mouse Movement Actions for available action names.

  <CodeGroup>
  ```toml Example
  [hints]
  auto_exit_actions = ["left_click", "middle_click"]
  ```
  </CodeGroup>

  <Warning>
    Including `move_mouse_relative` will cause the mode to exit on every arrow-key nudge, which is usually undesirable.
  </Warning>
</ParamField>

<ParamField path="hints.mode_exit_keys" type="array" default='[]'>
  Additional keys that exit hints mode (merged with `general.mode_exit_keys`). Must not conflict with `hint_characters`.
</ParamField>

<ParamField path="hints.hint_characters" type="string" default="asdfghjkl">
  Characters used to generate hint labels. Order matters for ergonomics. Must have at least 2 unique ASCII characters.

  <CodeGroup>
  ```toml Examples
  [hints]
  hint_characters = "asdfghjkl"    # Home row (default)
  hint_characters = "jkl;asdfg"    # Alternative home row
  hint_characters = "hjklasdfg"    # Vim-style
  ```
  </CodeGroup>
</ParamField>

<ParamField path="hints.backspace_key" type="string" default="Backspace">
  Key used for input correction. Must not conflict with `hint_characters` or action key bindings.

  <CodeGroup>
  ```toml Examples
  [hints]
  backspace_key = "Backspace"  # default
  backspace_key = "Delete"
  backspace_key = "Ctrl+H"
  ```
  </CodeGroup>
</ParamField>

<ParamField path="hints.mouse_action_refresh_delay" type="integer" default="0">
  Delay in milliseconds before refreshing hints after an action (0-10000). Useful for slow-loading apps or dynamic browser content.
</ParamField>

<ParamField path="hints.max_depth" type="integer" default="50">
  Maximum accessibility tree depth to traverse. Set to 0 for unlimited depth (may cause performance issues).
</ParamField>

<ParamField path="hints.parallel_threshold" type="integer" default="20">
  Minimum number of children to trigger parallel tree processing. Lower values enable more parallelization.
</ParamField>

### Visibility Options

<ParamField path="hints.include_menubar_hints" type="boolean" default="false">
  Show hints on menubar items.
</ParamField>

<ParamField path="hints.include_dock_hints" type="boolean" default="false">
  Show hints on Dock items.
</ParamField>

<ParamField path="hints.include_nc_hints" type="boolean" default="false">
  Show hints in Notification Center.
</ParamField>

<ParamField path="hints.include_stage_manager_hints" type="boolean" default="false">
  Show hints in Stage Manager.
</ParamField>

<ParamField path="hints.detect_mission_control" type="boolean" default="false">
  Auto-disable active app hints when in Mission Control. Only works on macOS 26+.

  <Warning>
    Do NOT enable on macOS versions before 26.x - causes false positives.
  </Warning>
</ParamField>

<ParamField path="hints.additional_menubar_hints_targets" type="array" default='["com.apple.TextInputMenuAgent", "com.apple.controlcenter", "com.apple.systemuiserver"]'>
  Extra menubar bundle IDs to include in hints.
</ParamField>

### Clickable Elements

<ParamField path="hints.clickable_roles" type="array" default='["AXButton", "AXComboBox", "AXCheckBox", "AXRadioButton", "AXLink", "AXPopUpButton", "AXTextField", "AXSlider", "AXTabButton", "AXSwitch", "AXDisclosureTriangle", "AXTextArea", "AXMenuButton", "AXMenuItem", "AXCell", "AXRow"]'>

</ParamField>

<ParamField path="hints.additional_ax_support.additional_electron_bundles" type="array" default='[]'>

  Default auto-detected bundles: `com.microsoft.VSCode`, `com.exafunction.windsurf`, `com.tinyspeck.slackmacgap`, `com.spotify.client`, `md.obsidian`
</ParamField>

<ParamField path="hints.additional_ax_support.additional_chromium_bundles" type="array" default='[]'>
  Custom Chromium browser bundle IDs to enable `AXEnhancedUserInterface` for.

  Default auto-detected bundles: `com.google.Chrome`, `com.brave.Browser`, `net.imput.helium`, `company.thebrowser.Browser`
</ParamField>

<ParamField path="hints.additional_ax_support.additional_firefox_bundles" type="array" default='[]'>
  Custom Firefox browser bundle IDs to enable `AXEnhancedUserInterface` for.

  Default auto-detected bundles: `org.mozilla.firefox`, `app.zen-browser.zen`
</ParamField>

### Basic Settings

<ParamField path="grid.enabled" type="boolean" default="true">

</ParamField>

<ParamField path="grid.auto_exit_actions" type="array" default='[]'>

  <CodeGroup>

  </CodeGroup>
</ParamField>

<ParamField path="grid.mode_exit_keys" type="array" default='[]'>
  Additional keys that exit grid mode (merged with `general.mode_exit_keys`).
</ParamField>

<ParamField path="grid.characters" type="string" default="abcdefghijklmnpqrstuvwxyz">
  Primary grid labels. Must have at least 2 unique ASCII characters.

  <CodeGroup>
  ```toml Examples
  [grid]
  characters = "abcdefghijklmnpqrstuvwxyz"  # Default

  ```
  </CodeGroup>
</ParamField>

<ParamField path="grid.sublayer_keys" type="string" default="abcdefghijklmnpqrstuvwxyz">

  Key to clear input and start over. Default is space.

  <CodeGroup>

</ParamField>

<ParamField path="grid.backspace_key" type="string" default="Backspace">
  Key for input correction.
</ParamField>

<ParamField path="grid.live_match_update" type="boolean" default="true">
  Highlight cells as you type for real-time feedback.
</ParamField>

<ParamField path="grid.hide_unmatched" type="boolean" default="true">
  Hide non-matching cells to reduce visual clutter.
</ParamField>

<ParamField path="grid.prewarm_enabled" type="boolean" default="true">
  Pre-compute grid on startup for faster first use (~1.5MB RAM).
</ParamField>

<ParamField path="grid.enable_gc" type="boolean" default="false">

## Recursive Grid Mode Configuration

Recursive grid divides the screen into cells, narrowing selection with each keypress.

### Basic Settings

<ParamField path="recursive_grid.enabled" type="boolean" default="true">
  Enable or disable recursive grid mode globally.
</ParamField>

<ParamField path="recursive_grid.auto_exit_actions" type="array" default='[]'>
  Actions that auto-exit recursive grid mode after execution.

  <CodeGroup>
  ```toml Example
  [recursive_grid]
  auto_exit_actions = ["left_click", "right_click"]
  ```
  </CodeGroup>
</ParamField>

<ParamField path="recursive_grid.mode_exit_keys" type="array" default='[]'>
  Additional keys that exit recursive grid mode (merged with `general.mode_exit_keys`).
</ParamField>

<ParamField path="recursive_grid.grid_cols" type="integer" default="2">
  Number of columns (minimum 2).
</ParamField>

<ParamField path="recursive_grid.grid_rows" type="integer" default="2">
  Number of rows (minimum 2).
</ParamField>

<ParamField path="recursive_grid.keys" type="string" default="uijk">
  Cell selection keys. Must have exactly `grid_cols × grid_rows` unique characters.

  <CodeGroup>
  ```toml Examples
  [recursive_grid]
  # 2×2 grid
  grid_cols = 2
  grid_rows = 2
  keys = "uijk"  # u=TL, i=TR, j=BL, k=BR

  # 3×3 grid
  grid_cols = 3
  grid_rows = 3
  keys = "gcrhtnmwv"
  ```
  </CodeGroup>
</ParamField>

<ParamField path="recursive_grid.backspace_key" type="string" default="Backspace">
  Key to go up one recursion level (backtrack).
</ParamField>

<ParamField path="recursive_grid.min_size_width" type="integer" default="25">
  Stop subdividing below this width in pixels (minimum 10).
</ParamField>

<ParamField path="recursive_grid.min_size_height" type="integer" default="25">
  Stop subdividing below this height in pixels (minimum 10).
</ParamField>

<ParamField path="recursive_grid.max_depth" type="integer" default="10">
  Maximum recursion levels (1-20).
</ParamField>

<ParamField path="recursive_grid.reset_key" type="string" default='" "'>
  Key to reset to initial center position. Default is space.
</ParamField>

## Font Configuration

Neru tries to find fonts in this order:

1. **PostScript name** (exact match) - e.g., `"SFMono-Bold"`, `"Menlo-Regular"`
2. **Family name** (looser matching) - e.g., `"SF Mono"`, `"JetBrains Mono"`
3. **Fallback** to system font

When `font_family = ""` (empty string):
- **Hints**: Bold system font
- **Grid/Scroll**: Regular system font

<CodeGroup>
```toml Monospace Fonts
[hints]

[hints.ui]
font_family = "SF Mono"
font_family = "Menlo"
font_family = "JetBrains Mono"
font_family = "Fira Code"
```

```toml PostScript Names
[hints]

[hints.ui]
font_family = "SFMono-Bold"
font_family = "Menlo-Bold"
```
</CodeGroup>

Find available fonts on macOS:

```bash
system_profiler SPFontsDataType | grep "Family:"
```

## Color Format

Neru colors use hex notation with optional alpha transparency.

### Supported Formats

| Format | Example | Alpha | Description |
|--------|---------|-------|-------------|
| `#AARRGGBB` | `#FF000000` | Yes | 8-char: Alpha + RGB (recommended) |
| `#RRGGBB` | `#FF0000` | No | 6-char: RGB only (fully opaque) |
| `#RGB` | `#F00` | No | 3-char shorthand |

### Format Breakdown: `#AARRGGBB`

| Position | Component | Range | Description |
|----------|-----------|-------|-------------|
| 1-2 | AA | 00-FF | Alpha (transparency) |
| 3-4 | RR | 00-FF | Red |
| 5-6 | GG | 00-FF | Green |
| 7-8 | BB | 00-FF | Blue |

### Alpha Channel Reference

| Opacity | Alpha | Use Case |
|---------|-------|----------|
| 100% | `FF` | Solid colors, high contrast |
| 95% | `F2` | Slight transparency |
| 90% | `E6` | Moderate transparency |
| 80% | `CC` | Noticeable transparency |
| 70% | `B3` | Semi-transparent (default for grid) |
| 60% | `99` | More transparent |
| 50% | `80` | Half transparent |
| 40% | `66` | Low visibility |
| 30% | `4D` | Subtle (default highlight) |
| 20% | `33` | Very subtle |
| 10% | `1A` | Barely visible |
| 0% | `00` | Invisible |

### Theme-Aware Colors

All color options use `_light` and `_dark` suffixes to support macOS Dark and Light Mode:

- **`*_light`** — used when macOS is in Light Mode
- **`*_dark`** — used when macOS is in Dark Mode

When these are not set in your config file (empty string `""`), Neru automatically uses sensible defaults that adapt to your system appearance. The colors update in real time when you switch system themes.

<CodeGroup>
```toml Custom Colors
[hints]

[hints.ui]
# Customizing only light mode background
background_color_light = "#FF0000AA"  # Custom red for light mode
# background_color_dark uses the built-in default
```
</CodeGroup>
