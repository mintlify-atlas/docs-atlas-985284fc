## Quick Diagnosis

Not working at all? Check these first:

```bash
# 1. Is daemon running?
neru status

# 2. Check permissions
neru doctor

# 3. Test basic functionality
neru hints  # Should show hints

# 4. Check logs
tail -20 ~/Library/Logs/neru/app.log
```

<Warning>
**Common quick fixes:**
- "Failed to connect to Neru daemon" → Daemon not running, run `neru launch`
- "Permission denied" → Grant accessibility permissions
- No hints appear → Check app exclusions, try different app
</Warning>

## Installation & Setup

<AccordionGroup>
  <Accordion title='"Cannot open Neru because the developer cannot be verified"'>
    Remove the quarantine attribute and open Neru:

    ```bash
    xattr -cr /Applications/Neru.app  # Remove quarantine
    open -a Neru
    ```
  </Accordion>

  <Accordion title='"Command not found: neru"'>
    Add Neru to your PATH:

    ```bash
    # Add to PATH
    export PATH="/usr/local/bin:$PATH"
    # Add to ~/.zshrc or ~/.bashrc
    ```
  </Accordion>

  <Accordion title="Homebrew installation fails">
    Update Homebrew and reinstall:

    ```bash
    brew update && brew reinstall --cask neru
    ```
  </Accordion>
</AccordionGroup>

## Hints & Grids

<AccordionGroup>
  <Accordion title="No hints or grids appear">
    Check these common causes:

    ```bash
    neru status              # Daemon running?
    neru doctor              # Permissions OK?
    neru hints               # CLI works?
    ```

    **Common fixes:**
    - Start daemon: `neru launch`
    - Grant permissions (see [Accessibility](/troubleshooting/accessibility))
    - Remove app from `excluded_apps` in config
    - Test in different app
  </Accordion>

  <Accordion title="Hints or grids are misaligned">
    This is rare. Enable debug logging and check logs:

    ```toml
    [logging]
    log_level = "debug"
    ```

    Then restart and monitor:

    ```bash
    pkill neru && neru launch
    tail -f ~/Library/Logs/neru/app.log
    ```

    <Note>
    If you encounter misalignment, please report with:
    - macOS version
    - App name and version
    - Screenshot
    </Note>
  </Accordion>

  <Accordion title="Hints don't appear in Electron apps">
    Electron apps need additional AX support.

    Edit `~/.config/neru/config.toml`:

    ```toml
    [hints.additional_ax_support]
    enable = true

    # If your app isn't auto-detected, add it:
    additional_electron_bundles = [
        "com.your.electronapp",
    ]
    ```

    Restart Neru:

    ```bash
    pkill neru && neru launch
    ```

    Check logs for:
    ```
    App requires Electron support
    Enabled AXManualAccessibility for: com.your.app
    ```
  </Accordion>

  <Accordion title="No hints in menubar or Dock">
    Enable menubar and Dock hints in your config:

    ```toml
    [hints]
    include_menubar_hints = true
    include_dock_hints = true

    # For specific menubar apps:
    additional_menubar_hints_targets = [
        "com.apple.controlcenter",
        "net.kovidgoyal.kitty",  # Example
    ]
    ```
  </Accordion>
</AccordionGroup>

## Hotkeys

<AccordionGroup>
  <Accordion title="Hotkey does nothing">
    **Possible causes:**
    1. Hotkey conflict with another app
    2. Daemon not running
    3. App is excluded
    4. Incorrect hotkey syntax

    **Solutions:**

    ```bash
    # 1. Test with CLI to bypass hotkey system
    neru hints

    # If CLI works, it's a hotkey issue

    # 2. Check daemon status
    neru status
    ```

    Try a different hotkey combination in `~/.config/neru/config.toml`:

    ```toml
    [hotkeys]
    "Ctrl+F" = "hints"  # Try this instead

    # Verify syntax is correct
    # Modifiers: Cmd, Ctrl, Alt/Option, Shift
    # Format: "Mod1+Mod2+Key" = "action"
    ```
  </Accordion>

  <Accordion title="Hotkey works in some apps but not others">
    The app is likely in your excluded list.

    Edit `~/.config/neru/config.toml`:

    ```toml
    [general]
    excluded_apps = [
        # "com.apple.Terminal",  # Comment out to enable
    ]
    ```

    Find an app's bundle ID:

    ```bash
    osascript -e 'id of app "AppName"'
    ```
  </Accordion>

  <Accordion title="Hotkey conflicts with system shortcuts">
    **Option 1: Change Neru hotkey**

    ```toml
    [hotkeys]
    "Cmd+Shift+Space" = ""  # Disable default
    "Ctrl+Alt+Space" = "hints"  # Use different combo
    ```

    **Option 2: Disable system shortcut**

    1. Open **System Settings → Keyboard → Keyboard Shortcuts**
    2. Find conflicting shortcut
    3. Disable or change it

    **Option 3: Use external hotkey manager**

    ```bash
    # Use skhd or similar instead of Neru hotkeys
    # ~/.config/skhd/skhdrc
    ctrl - f : neru hints
    ```

    Then disable Neru hotkeys:

    ```toml
    [hotkeys]
    # Leave empty or comment out all hotkeys
    ```
  </Accordion>
</AccordionGroup>

## Performance Issues

<AccordionGroup>
  <Accordion title="Hints appear slowly">
    **Possible causes:**
    1. Too many depth levels in the accessibility tree
    2. Debug logging enabled
    3. System resource constraints

    **Solutions:**

    ```bash
    # 1. Disable debug logging
    [logging]
    log_level = "info"  # Not "debug"

    # 2. Check system resources
    top -o cpu
    ```
  </Accordion>

  <Accordion title="High CPU usage">
    Neru should not use excessive CPU.

    ```bash
    # Check Neru CPU usage
    top -pid $(pgrep neru)

    # Check logs for errors
    tail -f ~/Library/Logs/neru/app.log | grep ERROR

    # Restart daemon
    pkill neru && neru launch
    ```
  </Accordion>
</AccordionGroup>

## Daemon Issues

<AccordionGroup>
  <Accordion title='"Failed to connect to Neru daemon"'>
    Daemon is not running.

    ```bash
    # Start daemon
    neru launch

    # Check status
    neru status

    # If still failing, check for stale socket
    # (path is printed in logs; typically under /var/folders/.../T)
    rm -f /var/folders/*/*/T/neru.sock
    neru launch
    ```
  </Accordion>

  <Accordion title="Daemon crashes on startup">
    Configuration error or system issue.

    ```bash
    # Check logs
    cat ~/Library/Logs/neru/app.log

    # Try with minimal config
    mkdir -p ~/.config/neru
    cat > ~/.config/neru/config.toml << EOF
    [hotkeys]
    "Cmd+Shift+Space" = "hints"

    [logging]
    log_level = "debug"
    EOF

    neru launch
    ```
  </Accordion>

  <Accordion title="Daemon stops responding">
    IPC socket issue or daemon hung.

    ```bash
    # Force quit
    pkill -9 neru

    # Clean up socket
    # (path is printed in logs; typically under /var/folders/.../T)
    rm -f /var/folders/*/*/T/neru.sock

    # Restart
    neru launch

    # Monitor logs
    tail -f ~/Library/Logs/neru/app.log
    ```
  </Accordion>

  <Accordion title="Daemon won't quit">
    Force termination needed.

    ```bash
    # Force quit
    pkill -9 neru

    # Or use Activity Monitor:
    # 1. Open Activity Monitor
    # 2. Search "Neru"
    # 3. Select and click "Force Quit"
    ```
  </Accordion>
</AccordionGroup>

## Configuration Issues

<AccordionGroup>
  <Accordion title="Config changes not taking effect">
    Daemon needs restart to reload config.

    ```bash
    # Restart daemon
    pkill neru && neru launch

    # Verify config location
    neru status
    # Check "Config:" line
    ```
  </Accordion>

  <Accordion title='"Failed to parse config"'>
    TOML syntax error.

    ```bash
    # Check logs
    cat ~/Library/Logs/neru/app.log | grep ERROR

    # Common issues:
    # - Missing quotes around keys/values
    # - Incorrect section headers
    # - Invalid TOML syntax

    # Validate TOML syntax online:
    # https://www.toml-lint.com/

    # Or use default config as reference:
    curl -o /tmp/default.toml \
      https://raw.githubusercontent.com/y3owk1n/neru/main/configs/default-config.toml
    ```
  </Accordion>

  <Accordion title="Colors not working">
    Check hex color format.

    ```toml
    # Correct:
    background_color = "#FFD700"

    # Incorrect:
    background_color = "FFD700"   # Missing #
    background_color = "#FFFGG"   # Invalid hex
    ```
  </Accordion>

  <Accordion title="Hotkeys in wrong format">
    Check modifier syntax.

    ```toml
    # Correct:
    "Cmd+Shift+Space" = "hints"

    # Incorrect:
    "Cmd-Shift-Space" = "hints"  # Use +, not -
    "CMD+SHIFT+SPACE" = "hints"  # Use proper case
    ```
  </Accordion>
</AccordionGroup>

## Keyboard Layout Issues

<AccordionGroup>
  <Accordion title="Wrong characters produced when typing">
    Neru supports most keyboard layouts including QWERTY, AZERTY, QWERTZ, Dvorak, and Colemak. Neru automatically detects your physical keyboard layout via macOS.

    If you're still experiencing issues:

    1. **Check your keyboard layout is properly configured in macOS:**
        - System Settings → Keyboard → Input Source
        - Ensure your desired layout is added and selected

    2. **Layout not detected correctly:**
        - Some custom layouts may not be detected properly
        - Try switching to a standard layout (US QWERTY, Colemak, Dvorak) in macOS input settings
  </Accordion>

  <Accordion title="Input methods not working (CJK IME)">
    Neru supports CJK input methods (Pinyin, Wubi, etc.). When using an input method:

    - Hints work correctly
    - Key presses are translated through your physical keyboard layout
    - The input method receives keys as expected

    If input methods still don't work:

    - Ensure the input method is properly installed and active in macOS
    - Check that Accessibility permissions are granted to Neru
  </Accordion>
</AccordionGroup>

## Logging and Debugging

<AccordionGroup>
  <Accordion title="Enable debug logging">
    Edit `~/.config/neru/config.toml`:

    ```toml
    [logging]
    log_level = "debug"
    ```

    Restart:

    ```bash
    pkill neru && neru launch
    ```
  </Accordion>

  <Accordion title="View logs">
    ```bash
    # Real-time monitoring
    tail -f ~/Library/Logs/neru/app.log

    # Last 100 lines
    tail -100 ~/Library/Logs/neru/app.log

    # Search for errors
    grep ERROR ~/Library/Logs/neru/app.log

    # Search for specific app
    grep "com.apple.Safari" ~/Library/Logs/neru/app.log
    ```
  </Accordion>

  <Accordion title="Common log messages">
    **"App requires Electron support"** - Electron app detected, needs AX support enabled

    **"Enabled AXManualAccessibility"** - Electron support activated successfully

    **"No clickable elements found"** - No hints to show (app may need custom config)

    **"Failed to get AX elements"** - Accessibility permission issue
  </Accordion>

  <Accordion title="Clear logs">
    ```bash
    # Remove old logs
    rm ~/Library/Logs/neru/app.log

    # Restart daemon (creates fresh log)
    pkill neru && neru launch
    ```
  </Accordion>
</AccordionGroup>

## Emergency Reset

<Warning>
If Neru is completely broken, perform a full reset:

```bash
# 1. Force quit
pkill -9 neru

# 2. Remove all Neru files
rm -rf /Applications/Neru.app
rm -f /usr/local/bin/neru
rm -rf ~/.config/neru
rm -rf ~/Library/Application\ Support/neru
rm -rf ~/Library/Logs/neru
# IPC socket lives under the OS temp directory
rm -f /var/folders/*/*/T/neru.sock

# 3. Reinstall
brew reinstall --cask neru
# or build from source

# 4. Fresh start (no config)
neru launch

# 5. Grant permissions again
# System Settings → Privacy & Security → Accessibility
```
</Warning>

## Getting Help

If none of these solutions work:

1. **Gather information:**
    - macOS version: `sw_vers`
    - Neru version: `neru --version`
    - App name and version where issue occurs
    - Config file (anonymize if needed)
    - Relevant logs

2. **Search existing issues:**
    - [GitHub Issues](https://github.com/y3owk1n/neru/issues)

3. **Open an issue:**
    - Include all gathered information
    - Describe expected vs actual behavior
    - Steps to reproduce

4. **Consider a PR:**
    - Pull requests are more likely to be reviewed than issues
    - Fix the problem yourself and contribute back
    - See [Development Guide](/development/building) for contribution guidelines
