## Overview

The Neru CLI provides comprehensive control over the daemon through a robust command-line interface. All commands communicate with the daemon via IPC (Inter-Process Communication) using Unix domain sockets, enabling fast, reliable communication and proper error handling.

<Card title="Key Features" icon="bolt">
  - Unix domain socket IPC for reliable communication
  - Multiple concurrent CLI commands supported
  - Structured error codes for scripting
  - Configurable timeouts
  - Command chaining and automation support
</Card>

## Daemon Control

### Launch

Start the Neru daemon:

```bash
neru launch
```

With custom config:

```bash
neru launch --config /path/to/config.toml
neru launch -c ./configs/author-config.toml
```

**Flags:**
- `--config, -c` - Path to custom configuration file
- `--timeout` - IPC timeout in seconds (default: 5)

### Start/Stop/Idle

```bash
neru start    # Resume paused daemon
neru stop     # Pause daemon (keeps running)
neru idle     # Cancel active mode
```

<Note>
  Use `stop` for temporary disable, `idle` to cancel the current navigation mode.
</Note>

### Status

Check daemon status and current mode:

```bash
neru status
```

**Output example:**
```
Neru Status:
  Status: running
  Mode: idle
  Config: /Users/username/.config/neru/config.toml
```

**Status values:** `running`, `stopped`  
**Mode values:** `idle`, `hints`, `grid`, `recursive_grid`, `scroll`

Show coordinate grid:

```bash

```

With immediate action:

Recursive cell-based navigation:


Vim-style scrolling:


### Action Flag Behavior

The `--action` flag allows commands to execute an action immediately upon selection:

<CodeGroup>
```bash Hints (Recommended)
neru hints --action right_click
# After typing hint label (e.g., "AA"), right-clicks automatically
```

```bash Grid
neru grid --action left_click
# Clicks after final grid selection
```

```bash Recursive Grid
neru recursive_grid --action left_click
# Clicks after reaching final depth
```
</CodeGroup>

<Tip>
  The `--action` flag is most useful for hints mode. For other modes, consider using `auto_exit_actions` in your config file instead.
</Tip>

### Mouse Movement

**Absolute positioning:**

```bash
neru action move_mouse --x 500 --y 300
```

**Center of screen:**

```bash
neru action move_mouse --center
```

**Center with offset:**

```bash
neru action move_mouse --center --x 50 --y -30
neru action move_mouse --center --x 100  # y defaults to 0
```

**Relative movement:**

```bash
neru action move_mouse_relative --dx 10 --dy -5
```

**Flags:**
- `--x` - X coordinate (pixels) or horizontal offset with `--center`
- `--y` - Y coordinate (pixels) or vertical offset with `--center`
- `--center` - Move to center of active screen
- `--dx` - Delta X for relative movement
- `--dy` - Delta Y for relative movement

## Service Management

### macOS (launchd)

```bash
neru services install    # Install and load launchd service
neru services status     # Check service status
```

Installs to `~/Library/LaunchAgents` for automatic startup on login.

<Note>
  If using Nix or Homebrew, use their respective service managers instead.
</Note>

## Screen Sharing

Toggle overlay visibility during screen sharing:

```bash
neru toggle-screen-share
```

**Behavior:**
- When hidden, overlay won't appear in shared screens but remains visible locally
- State resets to visible on restart
- Also accessible via system tray menu

<Warning>
  Effectiveness varies by macOS version and screen sharing application. Always test with your specific setup. Works reliably on macOS 14 and earlier.
</Warning>

## IPC Communication

### Protocol

The CLI communicates with the daemon using Unix domain sockets with JSON messages.

**Socket location:** `/var/folders/.../T/neru.sock` (typically in OS temp directory)

**Request format:**
```json
{
  "version": "1.0.0",
  "action": "hints",
  "params": {},
  "args": []
}
```

**Response format:**
```json
{
  "version": "1.0.0",
  "success": true,
  "message": "OK",
  "code": "OK",
  "data": {}
}
```

Source: [internal/core/infra/ipc/ipc.go:55-70](file:///home/daytona/workspace/source/internal/core/infra/ipc/ipc.go#L55-L70)

### Error Codes

Structured error codes for scripting:

| Code | Description |
|------|-------------|
| `OK` | Success |
| `ERR_UNKNOWN_COMMAND` | Invalid command |
| `ERR_NOT_RUNNING` | Daemon not running |
| `ERR_ALREADY_RUNNING` | Daemon already running |
| `ERR_MODE_DISABLED` | Mode disabled in config |
| `ERR_INVALID_INPUT` | Invalid input parameters |
| `ERR_ACTION_FAILED` | Action execution failed |

Source: [internal/core/infra/ipc/ipc.go:45-53](file:///home/daytona/workspace/source/internal/core/infra/ipc/ipc.go#L45-L53)

### Timeouts

```bash
# Default timeout (5 seconds)
neru status

# Custom timeout
neru --timeout 10 status
```

**Timeout constants:**
- Default timeout: 5 seconds
- Connection timeout: 2 seconds
- Ping timeout: 500ms
- Connection read timeout: 30 seconds

## Common Workflows

### Quick Navigation

```bash
# Start daemon
neru launch

# Use hints for clicking
neru hints
# Type hint label to click

# Right-click via hints
neru hints --action right_click

# Use grid for precise positioning
neru grid
```

### Status Monitoring

```bash
# Check if running
neru status &>/dev/null && echo "Running" || echo "Not running"

# Get current mode
neru status | grep "Mode:"
```

### Configuration Updates

```bash
# Edit config
vim ~/.config/neru/config.toml

# Reload without restart
neru config reload

# Verify changes
neru config dump
```

## Troubleshooting

### Command Hangs

```bash
# Force quit daemon
pkill -9 neru

# Restart
neru launch
```

### Socket Permission Errors

```bash
# Remove stale socket
rm -f /var/folders/*/*/T/neru.sock

# Restart daemon
neru launch
```

### Commands Not Working

```bash
# Verify daemon is running
neru status

# If not running, launch it
neru launch

# Check logs
tail -f ~/Library/Logs/neru/app.log
```
